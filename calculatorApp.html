Overview

I used Android studio to create this app. The professor said every calculator app he has ever looked over has had some sort of problem. So I made it my goal to make this app perfect with no flaws. I went through vigorous testing to ensure it was perfect and I succeeded achieving 100% for my final grade.

When creating my app, I decided to make my own combination of the Apple Calculator App and the Android Calculator App, combining my favorite details from both.

The one error I know exists and I didn't know how to fix is overfilling the textView with numbers. Also when my screen is rotated it doesn't save the calculation for some reason even though I set the id.



Here are all the edge cases that I covered (as far as I can remember) and how I handled them:
  • When clicking a number(not including 0) :
    • if the "=" button was just pressed
      • then the number pressed replaces the calculation
    • if there have been numbers input, and the user didn't type "0" after a decimal
      • then we remove the last char of the number and replace it with the clicked number
    • if there have been numbers input, and the only number on the screen is 0(even if the user input 0)
      • then we remove the last char of the number and replace it with the clicked number

  • When clicking an operator :
    • if an operator was the last thing input
      • then the new operator replaces the old one
    • if no numbers have been input before the operator
      • then it tells you that you can't do that

  • When clicking "0":
    • if the "=" button was just pressed
      • then the number pressed replaces the calculation
    • 0 cannot be placed after an operator
      • (otherwise I had a problem where I could input 00003 for example)
    • if there have been numbers input, and the user didn't type "0" after a decimal
      • then we remove the last char of the number and replace it with the clicked number
    • if there have been numbers input, and the only number on the screen is 0 (even if the user input 0)
      • then we remove the last char of the number and replace it with the clicked number
    • if 0 is the first number
      • other numbers being clicked replace

  • When clicking "=":
    • if the user clicks "=" when nothing has been input
      • nothing happens
    • if an operator was the last thing input
      • then an error message pops up

  • When clicking ".":
    • if the ArrayList "numbers" is empty
      • then it makes the first value 0.0
    • if the last element was an operator
      • it adds 0.0 for the next number
    • if the last number already has a decimal
      • then it sends an error message

  • When clicking "C":
    • if nothing is input
      • nothing happens
    • otherwise removes digits, one digit at a time
(Brackets, S and R buttons don't work cus I didn't need them, and I didn't have time left to finish implementing them, so they are incomplete.)
